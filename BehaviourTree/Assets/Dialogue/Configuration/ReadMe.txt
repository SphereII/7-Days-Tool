This folder will contain XML files that allow the Dialogue tool to add Actions and Requirements from other assemblies.

New actions and requirements can be added by adding a new xml file to the Source folder. The definition of the
XML file can be found below.

Once the file has been created, it needs to be added to the DialogEnums.cs under GeneratedFiles. 

Do not edit this file directly, as it will be over-written. It's generated based on all the Source files.

Close all dialogue nodes. Then, in the 7 Days To Die menu, click on Dialogue -> Admin -> Generate Enums.


[ Definition ]
The top level node does not matter.

Actions must be under the <actions> node.
Requirements must be under the <requirements> node.
Regardless if its an action or requirement, the properties are the same.

<score>
    <actions>
        <action name="Teleport" >
              <!-- The class + assembly -->
              <property name="class" value="AddCVar, SCore" />
              <!-- Description of what the action does -->
              <property name="description" value="Sets the specified cvar and value on the player." />
              
              <!-- Values -->
              <property name="valueDescription" value="CVar Value:" />
              <property name="valueType" value="float" />
              <property name="valueToolTip" value="The numeric value of the cvar" />
              
              <!-- ID -->
              <property name="idDescription" value="CVar Name:" />
              <property name="idType" value="string" />
              <property name="idToolTip" value="The name of the cvar" />
              
              <!-- Supported operator between Value and ID -->
              <property name="supportOperator" value="true" />
        </action>
    </actions>
    <requirements>
        <!-- Minimum example -->
        <requirement name="IsLeader" >
            <property name="class" value="Leader, SCore" />
            <property name="description" value="True if the player is the leader of the NPC." />
        </requirement>
    </requirements>
</score>
